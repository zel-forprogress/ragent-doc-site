AI 知识库建设#

知识库建设主流程#

创建知识库
  -> 上传文档
  -> 保存原始文件到 RustFS
  -> 保存文档记录到 PostgreSQL
  -> 发送 RocketMQ 分块消息
  -> 消费消息执行解析、分块、Embedding
  -> 保存 chunk
  -> 写入 t_knowledge_vector

关键接口#

功能 接口
创建知识库 POST /knowledge-base
上传文档 POST /knowledge-base/{kb-id}/docs/upload
开始分块 POST /knowledge-base/docs/{doc-id}/chunk
查看分块 GET /knowledge-base/docs/{doc-id}/chunks
查看日志 GET /knowledge-base/docs/{docId}/chunk-logs

实际前缀都是：

/api/ragent

普通分块模式#

对应 ProcessMode.CHUNK。

流程：

读取文件 -> Tika 解析 -> 分块策略 -> Embedding -> 写 chunk 和 vector

Pipeline 模式#

对应 ProcessMode.PIPELINE。

它把文档处理拆成节点：

FetcherNode -> ParserNode -> ChunkerNode -> EnhancerNode -> EnricherNode -> IndexerNode

核心引擎：

IngestionEngine

这个设计更适合扩展，例如以后加 OCR、表格增强、摘要增强、元数据抽取，都可以加节点。

AI 知识库建设#

知识库建设是 RAG 的“粮仓”。如果入库质量差，后面模型再强也很难答好。

知识库对象#

知识库表：

t_knowledge_base

核心字段：

  • name：知识库名称。
  • embedding_model：使用哪个 Embedding 模型。
  • collection_name：向量集合名。

对应服务：

KnowledgeBaseServiceImpl

文档上传#

文档表：

t_knowledge_document

上传接口：

POST /api/ragent/knowledge-base/{kb-id}/docs/upload

核心服务：

KnowledgeDocumentServiceImpl.upload

上传后不会立刻完成入库，而是先保存文档元信息和文件地址。

文件存储#

上传的原始文件存在 RustFS。

核心类：

S3FileStorageService

为什么用对象存储：

  • 数据库不适合保存大文件。
  • 文件和元数据分开，便于扩展。
  • RustFS 兼容 S3 API，开发环境部署简单。

文档解析#

Ragent 使用解析器把文件变成纯文本。

核心类：

类 说明
DocumentParser 解析接口
DocumentParserSelector 解析器选择
TikaDocumentParser 使用 Apache Tika 解析常见文档
MarkdownDocumentParser Markdown 解析
TextCleanupUtil 文本清洗

文档分块#

分块是 RAG 的关键环节。

核心接口：

ChunkingStrategy

已实现策略：

策略 类 说明
固定长度 FixedSizeTextChunker 按固定字符数切分
结构感知 StructureAwareTextChunker 尽量按标题、段落等边界切分

向量化#

分块之后要生成 Embedding。

核心类：

ChunkEmbeddingService
EmbeddingService
RoutingEmbeddingService

默认 Embedding 配置：

ai:
  embedding:
    default-model: qwen-emb-8b

异步入库#

点击“开始分块”后，Ragent 通过 RocketMQ 发消息：

knowledge-document-chunk_topic

消费类：

KnowledgeDocumentChunkConsumer

这样做的好处：

  • 上传接口不会卡很久。
  • 分块失败可以记录状态。
  • 入库任务可以异步重试和追踪。

Pipeline 入库#

除了普通分块模式，Ragent 还有 Pipeline 模式。

核心类：

  • IngestionEngine
  • IngestionNode
  • FetcherNode
  • ParserNode
  • ChunkerNode
  • EnhancerNode
  • EnricherNode
  • IndexerNode

Pipeline 的思想：

把文档处理拆成多个节点，每个节点只做一件事，节点之间通过上下文传递结果。

这比写死一个固定流程更灵活。
