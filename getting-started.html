从零启动 Ragent —— 前后端全流程#

中间件初始化#

完整依赖：

中间件 用途 默认地址
PostgreSQL + pgvector 业务数据和向量数据 127.0.0.1:15432
Redis Redisson、限流、锁 127.0.0.1:6379
RocketMQ 文档分块异步消息 127.0.0.1:9876
RustFS 文件对象存储 http://localhost:9000
MCP Server 工具调用 http://localhost:9099

启动前检查 Docker：

docker version
docker ps

如果报 Docker API 错误，先打开 Docker Desktop。

本地开发环境搭建#

需要：

  • JDK 17。
  • Node.js 18+。
  • Docker Desktop。
  • Maven Wrapper 使用项目自带 mvnw.cmd 即可。

检查：

java -version
node -v
npm -v
docker ps

Docker 中间件部署#

PostgreSQL：

docker run -d --name ragent-postgres `
  -e POSTGRES_DB=ragent `
  -e POSTGRES_USER=postgres `
  -e POSTGRES_PASSWORD=postgres `
  -p 15432:5432 `
  -v ragent-pgdata:/var/lib/postgresql/data `
  pgvector/pgvector:pg17

Redis：

docker run -d --name ragent-redis `
  -p 6379:6379 `
  redis:8.2.0 redis-server --requirepass 123456

RocketMQ：

docker compose -f .\resources\docker\rocketmq-stack-5.2.0.compose.yaml up -d

RustFS：

docker run -d --name rustfs_local `
  -p 9000:9000 `
  -p 9001:9001 `
  -v rustfs-data:/data `
  -e RUSTFS_ACCESS_KEY=rustfsadmin `
  -e RUSTFS_SECRET_KEY=rustfsadmin `
  rustfs/rustfs:latest `
  --console-enable `
  --access-key rustfsadmin `
  --secret-key rustfsadmin `
  /data

PostgreSQL 数据库初始化#

docker cp .\resources\database\schema_pg.sql ragent-postgres:/schema_pg.sql
docker cp .\resources\database\init_data_pg.sql ragent-postgres:/init_data_pg.sql
docker exec ragent-postgres psql -U postgres -d ragent -f /schema_pg.sql
docker exec ragent-postgres psql -U postgres -d ragent -f /init_data_pg.sql

验证：

docker exec ragent-postgres psql -U postgres -d ragent -c "select username, role from t_user;"

启动顺序#

第 1 个 PowerShell，MCP：

cd D:\git-localRepository\ragent
.\mvnw.cmd -pl mcp-server spring-boot:run

第 2 个 PowerShell，后端：

cd D:\git-localRepository\ragent
$env:BAILIAN_API_KEY="你的百炼Key"
$env:SILICONFLOW_API_KEY="你的SiliconFlow Key"
.\mvnw.cmd -pl bootstrap spring-boot:run

第 3 个 PowerShell，前端：

cd D:\git-localRepository\ragent\frontend
npm install
npm run dev

访问：

http://localhost:5174

当前你机器上已经遇到过的坑#

Redis 连不上#

错误：

Unable to connect to Redis server: 127.0.0.1:6379

原因通常是 Docker Desktop 没启动，或者 ragent-redis 没运行。

处理：

docker ps -a
docker start ragent-redis
docker exec ragent-redis redis-cli -a 123456 ping

PostgreSQL 连错本机库#

你机器本机 PostgreSQL 占用 5432，所以 Docker PostgreSQL 改成了：

15432 -> 5432

后端配置必须是：

url: jdbc:postgresql://127.0.0.1:15432/ragent?client_encoding=UTF8

前端端口冲突#

你机器 5173 被其他应用占用，所以项目已改成：

5174

从零启动 Ragent#

中间件初始化#

当前项目依赖这些中间件：

中间件 用途
PostgreSQL + pgvector 业务数据和向量数据
Redis 分布式锁、限流、缓存、Redisson
RocketMQ 文档分块异步任务、反馈消息
RustFS S3 兼容对象存储，保存上传文件
MCP Server 工具调用服务

注意：源码里还保留了 Milvus 配置和实现，但当前默认向量类型是 pg，首次启动不用部署 Milvus。

本地开发环境搭建#

需要安装：

  • JDK 17。
  • Docker Desktop。
  • Node.js 18+。
  • Git。
  • IDEA 或 VS Code。

检查命令：

java -version
docker version
node -v
npm -v

如果 docker version 报错，通常是 Docker Desktop 没启动。

Docker 中间件部署#

PostgreSQL#

你当前机器上本机 PostgreSQL 占用了 5432，所以 Docker PostgreSQL 建议映射到 15432：

docker run -d --name ragent-postgres `
  -e POSTGRES_DB=ragent `
  -e POSTGRES_USER=postgres `
  -e POSTGRES_PASSWORD=postgres `
  -p 15432:5432 `
  -v ragent-pgdata:/var/lib/postgresql/data `
  pgvector/pgvector:pg17

项目配置要对应：

spring:
  datasource:
    url: jdbc:postgresql://127.0.0.1:15432/ragent?client_encoding=UTF8
    username: postgres
    password: postgres

Redis#

docker run -d --name ragent-redis `
  -p 6379:6379 `
  redis:8.2.0 redis-server --requirepass 123456

验证：

docker exec ragent-redis redis-cli -a 123456 ping

返回 PONG 表示正常。

如果后端报：

Unable to connect to Redis server: 127.0.0.1:6379

通常是：

  • Docker Desktop 没启动。
  • ragent-redis 容器没启动。
  • 6379 没映射出来。

处理：

docker ps -a
docker start ragent-redis

RocketMQ#

仓库自带 compose：

docker compose -f .\resources\docker\rocketmq-stack-5.2.0.compose.yaml up -d

如果 8081 端口冲突，可以把 compose 里的端口映射改成：

- "18081:8081"
- "18082:8082"

然后重新启动：

docker compose -f .\resources\docker\rocketmq-stack-5.2.0.compose.yaml down
docker compose -f .\resources\docker\rocketmq-stack-5.2.0.compose.yaml up -d

RustFS#

docker run -d --name rustfs_local `
  -p 9000:9000 `
  -p 9001:9001 `
  -v rustfs-data:/data `
  -e RUSTFS_ACCESS_KEY=rustfsadmin `
  -e RUSTFS_SECRET_KEY=rustfsadmin `
  -e RUSTFS_CONSOLE_ENABLE=true `
  rustfs/rustfs:latest `
  --console-enable `
  --access-key rustfsadmin `
  --secret-key rustfsadmin `
  /data

项目配置：

rustfs:
  url: http://localhost:9000
  access-key-id: rustfsadmin
  secret-access-key: rustfsadmin

PostgreSQL 数据库初始化#

复制脚本进容器：

docker cp .\resources\database\schema_pg.sql ragent-postgres:/schema_pg.sql
docker cp .\resources\database\init_data_pg.sql ragent-postgres:/init_data_pg.sql

执行初始化：

docker exec ragent-postgres psql -U postgres -d ragent -f /schema_pg.sql
docker exec ragent-postgres psql -U postgres -d ragent -f /init_data_pg.sql

验证用户：

docker exec ragent-postgres psql -U postgres -d ragent -c "select username, role from t_user;"

应该看到：

admin | admin

启动 MCP 服务#

新开一个 PowerShell：

cd D:\git-localRepository\ragent
.\mvnw.cmd -pl mcp-server spring-boot:run

看到 Tomcat started on port 9099 表示成功。

启动后端服务#

如果要正常问答，先配置 API Key：

$env:BAILIAN_API_KEY="你的百炼Key"
$env:SILICONFLOW_API_KEY="你的SiliconFlow Key"

然后启动：

cd D:\git-localRepository\ragent
.\mvnw.cmd -pl bootstrap spring-boot:run

成功标志：

Tomcat started on port 9090 with context path '/api/ragent'

启动前端#

你当前前端端口已改成 5174：

cd D:\git-localRepository\ragent\frontend
npm install
npm run dev

访问：

http://localhost:5174
