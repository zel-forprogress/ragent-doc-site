RAG 项目经验如何写进简历#

简历不要只写“接入大模型”#

写“接入大模型 API 实现智能问答”太薄了，因为这只说明你会调用接口。RAG 项目的简历重点应该是“工程链路”。

你可以按 4 层写：

层次 简历关键词
数据层 文档解析、文本清洗、分块策略、Embedding、向量入库
检索层 pgvector、相似度检索、多通道检索、去重、Rerank
生成层 Prompt 工程、上下文拼接、流式输出、会话记忆
工程层 Redis 限流、RocketMQ 异步任务、S3 对象存储、Trace、模型路由降级

可直接使用的简历描述#

版本一，适合写项目经历：

基于 Java 17 + Spring Boot 3 + React + PostgreSQL/pgvector + Redis + RocketMQ 实现企业级 RAG 智能问答系统，支持知识库管理、文档解析分块、Embedding 向量化、多通道检索、MCP 工具调用、会话记忆、模型路由降级和全链路 Trace。

版本二，适合强调问答链路：

负责 RAG 问答链路设计：用户问题进入系统后，依次经过会话历史加载、问题改写与拆分、意图树识别、模糊意图澄清、多通道并行检索、结果去重与重排、Prompt 组装、模型流式输出，并通过 Trace 记录每个节点耗时和中间结果。

版本三，适合强调知识库建设：

负责知识库入库链路：使用 RustFS 保存原始文件，基于 Apache Tika 抽取文档文本，支持结构感知分块和固定长度分块，调用 Embedding 模型生成向量，写入 PostgreSQL pgvector，并通过 RocketMQ 异步执行入库任务。

版本四，适合强调稳定性：

设计多模型路由与容错机制，将 Chat、Embedding、Rerank 按能力拆分为候选模型列表，结合优先级、健康状态和深度思考能力选择模型，调用失败后自动切换候选模型，提高模型服务可用性。

面试时如何讲项目架构#

可以按这个顺序讲：

  • 前端是 React + Vite，负责问答页和管理后台。
  • 后端是 Spring Boot 多模块单体，业务集中在 bootstrap。
  • 数据存储使用 PostgreSQL，向量检索使用 pgvector。
  • Redis 用于限流、锁和 Redisson。
  • RocketMQ 用于文档分块这种耗时任务的异步化。
  • RustFS 用于保存上传文件。
  • infra-ai 屏蔽百炼、SiliconFlow、Ollama 等模型供应商差异。
  • mcp-server 模拟外部业务工具，主服务通过 MCP 协议调用它。

面试官追问时的回答模板#

问题：你这个项目和普通 ChatGPT 套壳有什么区别？

回答：

普通套壳项目通常只是把用户输入转发给大模型，模型只能基于已有知识回答。Ragent 是完整 RAG 链路，先把企业文档解析、分块、向量化并写入知识库；用户提问时会做会话记忆、问题改写、意图识别和多通道检索，再把检索内容组装进 Prompt 给模型。它还包含 MCP 工具调用、模型路由降级和 Trace 排障，更接近企业应用。
