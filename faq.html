常见启动错误速查#

Docker API 连接失败#

错误：

failed to connect to the docker API at npipe:////./pipe/dockerDesktopLinuxEngine

原因：Docker Desktop 没启动。

处理：打开 Docker Desktop，等左下角变成 Running，再执行 docker ps。

后端连不上 Redis#

错误：

Unable to connect to Redis server: 127.0.0.1:6379

处理：

docker ps -a
docker start ragent-redis
docker exec ragent-redis redis-cli -a 123456 ping

后端数据库密码错误#

错误：

Password authentication failed for user "postgres"

你的机器上曾经出现过这个问题，原因是本机 PostgreSQL 占用了 5432，后端误连本机库。

解决方案：Docker PostgreSQL 映射到 15432，并把配置改为：

url: jdbc:postgresql://127.0.0.1:15432/ragent?client_encoding=UTF8

RocketMQ Dashboard 打不开#

错误：

ERR_CONNECTION_REFUSED
cannot join network namespace of a non running container

原因：rmqbroker 没起来，常见是 8081 端口冲突。

处理：修改 compose 端口映射，例如：

- "18081:8081"
- "18082:8082"

前端 5173 端口冲突#

你当前已经改为：

server: {
  port: 5174
}

访问：

http://localhost:5174
