AI 知识问答#

用户提问后发生了什么#

完整链路：

前端 ChatInput
  -> chatService 发起 SSE
  -> RAGChatController
  -> RAGChatServiceImpl
  -> StreamChatPipeline
  -> LLMService 流式输出
  -> 前端 MessageItem 增量渲染

StreamChatPipeline 逐行理解#

StreamChatPipeline.execute 是问答链路最值得读的代码。它的步骤：

loadMemory(ctx)
rewriteQuery(ctx)
resolveIntents(ctx)
handleGuidance(ctx)
handleSystemOnly(ctx)
retrieve(ctx)
handleEmptyRetrieval(ctx, retrievalCtx)
streamRagResponse(ctx, retrievalCtx)

翻译成人话：

  • 把历史对话拿出来。
  • 把用户问题改写成更完整的问题。
  • 判断问题属于哪些意图。
  • 如果问题太模糊，先反问用户。
  • 如果是系统类回答，不检索知识库。
  • 否则进入检索。
  • 如果没检索到，直接提示没有相关内容。
  • 如果检索到，组装 Prompt 并调用模型。

问答链路中的关键设计#

设计 作用
@ChatRateLimit 控制并发问答，避免模型被打爆
RagTraceNode 记录 RAG 节点
StreamTaskManager 支持停止生成
GuidanceDecision 问题模糊时先澄清
RetrievalContext 聚合知识库和 MCP 上下文

AI 知识问答链路#

Ragent 的问答入口是：

GET /api/ragent/rag/v3/chat

核心执行类是：

RAGChatServiceImpl -> StreamChatPipeline

StreamChatPipeline 的执行顺序可以概括为：

  • 加载会话记忆。
  • 问题改写和拆分。
  • 意图识别。
  • 判断是否需要澄清。
  • 判断是否是纯系统回答。
  • 检索知识库或调用 MCP。
  • 如果检索为空，返回提示。
  • 构建 Prompt。
  • 调用大模型流式输出。

对应代码逻辑：

loadMemory(ctx)
rewriteQuery(ctx)
resolveIntents(ctx)
handleGuidance(ctx)
handleSystemOnly(ctx)
retrieve(ctx)
handleEmptyRetrieval(ctx, retrievalCtx)
streamRagResponse(ctx, retrievalCtx)

会话记忆#

会话记忆的目的：让模型知道“用户前面说过什么”。

核心类：

  • ConversationMemoryService
  • DefaultConversationMemoryService
  • JdbcConversationMemoryStore
  • JdbcConversationMemorySummaryService

数据表：

  • t_conversation
  • t_message
  • t_conversation_summary

问题改写#

用户问题常常不适合直接检索，比如：

它怎么申请？

系统需要结合上下文改写成：

员工报销流程如何申请？

对应类：

  • QueryRewriteService
  • MultiQuestionRewriteService
  • RewriteResult

意图识别#

意图识别用于判断问题属于哪个业务方向：

  • 是查知识库？
  • 是调 MCP 工具？
  • 是普通系统回答？
  • 是需要澄清？

核心类：

  • IntentResolver
  • DefaultIntentClassifier
  • IntentTreeFactory
  • IntentNodeRegistry
  • IntentGuidanceService

数据表：

t_intent_node

多通道检索#

Ragent 不是只做单一路径向量检索，而是有多通道检索引擎：

MultiChannelRetrievalEngine

它会收集所有启用的 SearchChannel，并行执行，再走后处理器链。

已实现的通道：

通道 类 说明
全局向量检索 VectorGlobalSearchChannel 在相关知识空间中做向量相似度检索
意图定向检索 IntentDirectedSearchChannel 根据意图节点定向检索对应知识库

后处理器：

后处理器 类 说明
去重 DeduplicationPostProcessor 去掉重复 chunk
重排 RerankPostProcessor 调用 Rerank 模型重新排序

Prompt 构建#

检索到内容后，需要把它组织成模型能理解的 Prompt。

核心类：

  • RAGPromptService
  • PromptContext
  • DefaultContextFormatter
  • PromptTemplateLoader

Prompt 通常包含：

  • 用户问题。
  • 改写后的问题。
  • 知识库上下文。
  • MCP 工具结果。
  • 意图信息。
  • 历史对话。

流式输出#

Ragent 使用 SSE 给前端输出答案。

核心类：

  • StreamCallback
  • StreamChatEventHandler
  • SseEmitterSender
  • StreamTaskManager

前端对应：

  • frontend/src/hooks/useStreamResponse.ts
  • frontend/src/services/chatService.ts
  • frontend/src/components/chat/MessageItem.tsx
